# 事前验尸法

## 核心原理

事前验尸由加里·克莱因等人推广，其操作是：在项目尚未失败前，团队被正式要求设想「已经到了复盘日，且项目已惨败」，然后各自独立写下失败原因，再汇总排序。心理学上，它利用「 prospective hindsight」降低过度乐观与计划谬误：当人们被允许在心理上「已经失败」，防御性自我形象对负面想象的抑制会减弱，从而更愿意说出真实担忧。与事后复盘不同，验尸发生在资源仍可支配的时点，能把「可能的问题」转化为预防设计、触发指标与应急预算。成功的关键在于心理安全与匿名贡献（至少初稿阶段），以及把原因翻译成可验证的前置信号和负责人，而不是停留在「我们会更努力」的口号层。

## 使用步骤

1. **宣读规则**：领导者明确「此刻假设项目已失败」，禁止当场反驳他人，鼓励具体机制而非人身攻击。
2. **独立书写**：限时 5～10 分钟，每人写下尽可能多的失败原因，从执行、市场、技术、组织、外部冲击等维度发散。
3. **轮询汇总**：白板归类去重，按频率与严重性初步聚类，标出「高影响且高共识」项。
4. **反推时间线**：为 top 风险倒推最早出现的征兆（如关键岗位空缺超 2 周、某 KPI 连续三周下滑）。
5. **设计预防与缓冲**：为每条 top 风险指定预防动作、触发阈值、应急资源与决策人。
6. **嵌入节奏**：把触发器写进周报、里程碑评审或风险登记册，指定再评估日期。

## 使用场景与案例

### 场景一：新产品六个月上线
**背景**：团队士气高，路线图紧凑，但从未做过同类集成。
**应用**：验尸列出第三方 API 限流、核心开发病假、合规审查拖延等；征兆包括联调环境不稳定、法务问题单积压；预防为双供应商接口抽象、知识文档化、提前并行合规咨询。
**结果**：实际中途出现 API 变更，因预案缩短延期至两周而非两个月。

### 场景二：大型会议与发布会
**背景**：市场部策划千人活动，供应商多、天气不可控。
**应用**：验尸写出直播中断、嘉宾缺席、签到拥堵；触发为彩排时网络抖动次数、交通预警；预防含备用线路、替补演讲、分时段入场。
**结果**：活动日短时断网，备用线路切换，舆情可控。

### 场景三：并购后系统割接
**背景**：IT 需在周末完成核心 ERP 切换，业务停服窗口极短。
**应用**：验尸列出数据不一致、回滚脚本未测、关键用户未培训；预防为蓝绿演练、冻结非紧急需求、战时指挥架构与回滚决策授权。
**结果**：割接当晚出现一条映射错误，按预案回滚子模块而非全系统崩溃。

### 场景四：博士生论文投稿顶会
**背景**：截稿前四周，实验尚未完全收敛。
**应用**：验尸写出实验不可复现、审稿人质疑基线、图表错误；预防为预注册关键实验、内部 replicate、预留一周润色与校对轮次。
**结果**：审稿意见虽尖锐，但有完整附录回应，录用概率提高。

### 场景五：社区团购冷启动
**背景**：创始人计划三个月覆盖十个小区，地推预算有限。
**应用**：验尸列出团长流失、履约差评、竞品补贴；征兆为次日复购率、履约准时率；预防为团长培训与激励阶梯、小范围试点后再扩区。
**结果**：先深耕三小区打透模型，避免资金烧在无效扩张。

## 常见误区

- 开成「秋后算账」预演，变成对个人道德的指责，破坏心理安全。
- 只列宏大外部因素（「市场不好」）无可操作预防动作。
- 汇总后无负责人与截止日期，验尸沦为一次性发泄。
- 领导当场否定「不可能发生」，压制了最有价值的声音。
- 与乐观预算完全脱节，预防资源从未进入基线计划。

## 搭配工具

- **鱼骨图**：对高频失败原因做因果分类，补全人机料法环测维度。
- **SWOT**：把验尸识别的威胁与内部劣势交叉，形成 WT/WO 应对主题。
- **决策矩阵**：在有限预算下对预防措施按影响×成本排序。
- **第一性原理**：区分「结构性必败」与「执行性可防」，避免用战术勤奋掩盖战略错误。
