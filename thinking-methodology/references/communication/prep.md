# PREP 结构

## 核心原理

PREP 是日式商务沟通中常用的短表达框架：**Point（观点）**开门见山给出立场或结论；**Reason（原因）**用 1～3 条逻辑或价值判断解释为何如此；**Example（举例）**用数据、案例或类比使抽象原因可感知；最后 **Point（重申）**以不同措辞回扣观点，强化记忆。其原理与认知负荷有关：听众先获得「答案锚点」，后续信息会被主动归类到该锚点下；首尾双 Point 形成「框架效应」，降低中途走神导致的误解。PREP 适合会议发言、面试、电梯间偶遇领导等**时间受限**场景，可单独使用，也可作为金字塔原理的口语压缩版。

## 使用步骤

1. **用一句话写清 Point**：避免「可能、也许」，必要时加范围（如「在本季度内」）。
2. **筛选 Reason**：最多三条，每条一句；删除弱相关理由，优先因果与利益相关方。
3. **为每条 Reason 配 Example 或合并举例**：数字优于形容词；故事优于口号。
4. **设计重申句**：与首句 Point 同义转述，可加上行动呼吁或下一步。
5. **计时演练**：目标 60～90 秒内讲完，删繁就简。

## 使用场景与案例

### 场景一：站会上的阻塞说明
**背景**：每日站会每人一分钟。
**应用**：Point「今天我无法合入分支」；Reason「依赖服务契约未冻结」；Example「对方 API 昨天又改字段」；Point「需要接口负责人在下班前确认 v1 契约」。
**结果**：负责人立刻知道要介入什么，而非听完现象再猜诉求。

### 场景二：向配偶提议周末安排
**背景**：双方时间紧，易吵成情绪战。
**应用**：Point「我建议周六上午亲子馆、下午各自休息」；Reason「孩子社交需要、我们也都缺独处」；Example「上周全天连轴后周日都头疼」；Point「所以这样排对全家恢复最好」。
**结果**：把争论拉回方案比较，减少指责。

### 场景三：销售初次拜访后的跟进电话
**背景**：客户忙，只给两分钟。
**应用**：Point「建议下周安排 POC」；Reason「贵司痛点与我们的产品路径匹配、IT 已认可安全基线」；Example「同行业 X 在两周内把工单时长降了 30%」；Point「POC 是验证匹配度最快的方式」。
**结果**：推进下一步有清晰钩子。

### 场景四：学生对导师申请延期提交
**背景**：导师邮件简短，需高效回复。
**应用**：Point「请求延期三天」；Reason「实验重复方差未达标、需补一轮」；Example「目前三组里两组 p 值边缘」；Point「三天内可提交可复现数据与修订稿」。
**结果**：导师易判断合理性并给底线。

### 场景五：内部推动改用新工具
**背景**：午餐桌 informal 游说。
**应用**：Point「我们组该统一用新 CI」；Reason「旧流水线排队长、日志不可追溯」；Example「上周发布回滚多花四小时定位」；Point「换 CI 是短期学习成本换长期确定性」。
**结果**：同伴带走一个可复述的论点。

## 常见误区

- Reason 与 Point 无因果关系，只有情绪宣泄。
- Example 过长占满时间，Reason 被挤掉。
- 首尾 Point 完全重复同一句，显得机械；应同义转述。
- 多条 Reason 彼此矛盾未被发现，削弱可信度。

## 搭配工具

- **金字塔原理**：PREP 用于口头，书面可展开为完整金字塔。
- **RIDE**：说服场景下 Reason 可嵌入风险与利益维度。
- **电梯演讲**：时间更短时只保留 P-R-简例-P。
