# 播客脚本 / Podcast Script - 内容分类

## 定位

专为音频内容设计的脚本。播客是"耳朵经济"的核心载体，强调声音的魅力、故事的沉浸感和陪伴感。

## 写作目标

- **留存收听**：前60秒建立期待，让人继续听
- **陪伴感营造**：像朋友聊天，而非单向灌输
- **场景适配**：通勤、运动、睡前等多场景可用
- **记忆点打造**：听完能带走1-3个核心观点

## 核心特征

### 播客 vs 视频脚本
| 维度 | 播客脚本 | 视频脚本 |
|------|----------|----------|
| **感官通道** | 仅听觉 | 听觉+视觉 |
| **注意力** | 伴随式，可做多任务 | 独占式，需专注观看 |
| **节奏** | 更慢，留白更多 | 更快，信息密集 |
| **结构** | 线性叙事，环环相扣 | 模块化，可跳跃 |
| **互动** | 通过声音建立亲密感 | 通过画面建立冲击力 |

### 时长与结构
| 播客类型 | 时长 | 结构特点 | 适合内容 |
|----------|------|----------|----------|
| 短播客 | 10-20分钟 | 单一主题，开门见山 | 新闻解读、工具推荐 |
| 标准播客 | 30-45分钟 | 引入-展开-总结 | 深度访谈、专题讨论 |
| 长播客 | 60-90分钟 | 多板块，有中场过渡 | 长篇故事、系列对谈 |

### 核心指标
| 评级 | 指标 | 说明 |
|------|------|------|
| Ⅴ | 完听率 | 听完节目的比例，>40%为优秀 |
| Ⅴ | 订阅转化 | 试听转订阅的比例 |
| Ⅳ | 收听时长 | 平均每次收听时长 |
| Ⅳ | 分享率 | 播客被分享的频率 |
| Ⅲ | 评论互动 | 音频内容获得文字评论较少 |

## 标准结构模板

### 单人播客结构（30分钟）

```
【片头 0-1分钟】
- 音乐 + 节目标识音
- 本期内容预告
- "欢迎来到XX播客，我是XX"

【开场 1-5分钟】
- 问候听众，建立连接
- 本期主题引入（故事/问题/现象）
- 价值承诺："听完这期，你会了解..."

【主体 5-25分钟】
- 分3-4个板块展开
- 每板块间有过渡句
- 穿插案例、数据、故事

【结尾 25-29分钟】
- 核心观点总结
- 行动建议或思考题
- 下期预告

【片尾 29-30分钟】
- 订阅引导
- 联系方式
- 片尾音乐
```

### 访谈播客结构（45-60分钟）

```
【开场 0-5分钟】
- 主持人介绍嘉宾
- 嘉宾背景故事简述
- 本期话题引入

【破冰 5-10分钟】
- 轻松话题开场
- 建立对话氛围
- 展示嘉宾个性

【核心讨论 10-45分钟】
- 3-5个核心问题
- 每个问题深入追问
- 允许话题自然流动

【总结 45-55分钟】
- 嘉宾给听众的建议
- 资源推荐
- 未来计划

【结束 55-60分钟】
- 感谢嘉宾
- 订阅引导
- 下期预告
```

### 脚本格式规范

```markdown
# 播客标题

## 基础信息
- 时长：XX分钟
- 类型：[单人/访谈/对谈]
- 目标受众：

## 脚本正文

### 【片头】（0:00-1:00）
**BGM：** [音乐名称]
**台词：** 
"欢迎来到《XX播客》，我是主播XX。今天我们要聊的是..."

**音效提示：** 淡入，持续10秒后渐弱

---

### 【开场】（1:00-5:00）
**台词：**
"最近我发现一个有趣的现象..."

**语气：** 轻松、亲和
**停顿：** [停顿2秒] 留给听众思考
**强调：** 关键词放慢语速加重音

---

### 【主体板块一】（5:00-15:00）
...

## 制作备注
- 音乐清单：
- 音效需求：
- 剪辑要点：
```

## 语言风格

### 声音友好原则
| ❌ 不适合 | ✅ 适合 |
|-----------|--------|
| 复杂长句 | 短句+停顿 |
| 视觉描述("看这个图") | 听觉描述("想象一下") |
| 数字堆砌 | 转化为故事 |
| 专业术语（不解释） | 类比解释 |

### 亲密感营造
- **直接称呼**："你可能会想..." "你有没有过..."
- **分享脆弱**："我也曾经..." "说实话..."
- **留白艺术**：重要观点后停顿2-3秒
- "笑声和语气词"：自然的"嗯""啊""对"增加真实感

### 叙事节奏
- **钩子密度**：每5-8分钟一个小高潮
- **信息节奏**：干货-故事-干货-故事交替
- **情绪曲线**：有起伏，避免平铺直叙

## 平台适配

| 平台 | 特点 | 脚本注意 |
|------|------|----------|
| **小宇宙** | 中文播客主阵地，社区氛围好 | 注重节目shownotes |
| **Apple Podcasts** | 国际用户，品质要求高 | 英文介绍，专业制作 |
| **Spotify** | 算法推荐强，发现机制好 | 标签、分类要精准 |
| **喜马拉雅** | 用户基数大，泛知识类受欢迎 | 可以更长，更故事化 |

## Shownotes（节目笔记）写作

### 作用
- SEO优化，便于搜索发现
- 提供节目中提到的链接和资源
- 时间戳导航，方便跳转

### 结构
```markdown
# 节目标题

## 本期简介
2-3句话概括本期内容

## 时间戳
- 00:01:23 话题A
- 00:05:40 话题B
- 00:15:30 话题C

##  mentioned
- 书籍：《书名》
- 工具：工具名
- 链接：https://...

## 主播
- 主播名：简介
- 联系方式：

## 订阅
- Apple Podcasts
- 小宇宙
- Spotify
```

## 常见错误

| 评级 | 错误 | 示例 | 修正 |
|------|------|------|------|
| Ⅰ | 视觉依赖 | "看这个图表" | "让我描述一下这个趋势..." |
| Ⅰ | 节奏过快 | 密集信息无停顿 | 每段后留白 |
| Ⅰ | 缺乏过渡 | 话题跳跃突兀 | "说到这里，我想到了..." |
| Ⅱ | 过于 scripted | 照本宣科无生气 |  bullet point + 即兴发挥 |
| Ⅱ | 开头太绕 | 5分钟还没进入主题 | 60秒内建立期待 |
