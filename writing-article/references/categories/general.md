# 通用内容 - 内容分类

## 定位

观点类、知识类、叙述类的常规文章。无特定格式约束，当类型不明确时默认加载。

## 写作目标

让读者读完后有所得：有所了解、有所思考、或有所改变。

## 核心特征

### 内容定位
- **主题泛化**：不绑定特定行业或场景，以普适性知识传递为主
- **读者广泛**：面向对该主题感兴趣的普通读者，而非特定职业群体
- **时效适中**：内容半衰期较长，不追逐热点，但保持与当下语境的相关性

### 结构自由度
| 评级 | 特征说明 |
|------|---------|
| Ⅴ | 无固定格式约束，根据主题自然展开 |
| Ⅴ | 支持多种文体：论述、叙事、说明、随笔 |
| Ⅳ | 1500字以上建议分节，增强可读性 |
| Ⅱ | 过度使用小标题导致文章碎片化 |
| Ⅰ | 强行套用不适合的结构模板 |

### 语言调性
- **理性与温度并存**：逻辑清晰，但不冰冷
- **第二人称优先**：直接与读者对话，拉近距离
- **适度修辞**：比喻、排比等手法服务于理解，而非炫技

## 默认结构

```
开篇  →  建立问题或场景，让读者感知"这和我有关"
正文  →  展开论证 / 叙述 / 分析，分 2～4 个层次
结尾  →  收束观点，给出余味、行动指引或升华
```

## 语言风格

- 理性中带温度，不说教，不过度煽情
- 长短句结合，有节奏
- 第一或第二人称拉近读者距离

## 默认字数

| 类型 | 建议字数 |
|------|---------|
| 短文 | 500～1000 字 |
| 常规文章 | 1000～2000 字 |
| 深度长文 | 2000～4000 字 |

## 结尾类型（可选一种）

- **提问**：将核心矛盾抛给读者，引发思考
- **点睛**：一句话升华全文
- **行动**：告诉读者读完可以做什么
- **余味**：意在言外，留白给读者

## 排版

使用 Markdown。标题分层（`##` / `###`），1500 字以上才考虑小标题。
