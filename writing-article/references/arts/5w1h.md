# 5W1H 框架 (Who-What-When-Where-Why-How) - 写作技巧

## 什么是 5W1H

**5W1H** 是一个基于六个基本问题的信息收集和内容组织框架，代表：
- **Who**（谁）
- **What**（什么）
- **When**（何时）
- **Where**（何地）
- **Why**（为什么）
- **How**（如何）

它起源于新闻业，是确保报道完整性的标准检查清单。如今，5W1H 已广泛应用于内容创作、项目管理、问题分析、故事构建等多个领域，被誉为"万能思维工具"。

## 理论溯源

### 起源：新闻业的"黄金法则"

**Rudyard Kipling 的贡献**：

5W1H 的概念最早由英国著名作家 **Rudyard Kipling**（《丛林之书》作者）在 1902 年的著作 *Just So Stories* 中通过一首诗歌系统阐述：

> "I keep six honest serving-men  
> (They taught me all I knew);  
> Their names are What and Why and When  
> And How and Where and Who."

这首诗形象地描述了六个"诚实的仆人"（问题）如何帮助他探索和理解世界。

### 新闻业的标准化

20 世纪初，5W1H 被新闻学院采纳为**新闻写作的核心框架**：
- **要求**：新闻报道的开篇（导语/Lead）必须回答 5W1H 的所有问题
- **目的**：确保读者在首段就能获得事件的完整概览
- **别称**：被称为"新闻六要素"或"倒金字塔结构的基石"

**现代新闻教育中的 5W1H**：

| 要素 | 问题 | 新闻导语示例 |
|------|------|-------------|
| **Who** | 谁涉及？ | "一名45岁的银行劫匪..." |
| **What** | 发生了什么？ | "...抢劫了市中心珠宝店..." |
| **When** | 何时发生？ | "...于周二午餐时间..." |
| **Where** | 何地发生？ | "...位于第五大道..." |
| **Why** | 为什么发生？ | "...试图获取毒资..." |
| **How** | 如何发生？ | "...用香蕉假装枪支..." |

### 跨领域扩展

5W1H 的应用已远远超出新闻业：

**时间线**：
- **1902年**：Kipling 诗歌起源
- **1920-1950s**：新闻学院标准化教学
- **1960-1980s**：质量管理（TQM）引入制造业
- **1990-2000s**：项目管理、咨询业广泛采用
- **2010s-现在**：内容营销、数据叙事、AI 提示工程

### 三层应用模式

#### 1. 信息收集层（调查阶段）

**用途**：确保获取所有必要信息，无遗漏。

**示例**（采访场景）：
- Who: "谁参与了这个项目？"
- What: "项目的具体目标是什么？"
- When: "时间表是怎样的？"
- Where: "在哪个市场/地区执行？"
- Why: "为什么要做这个项目？"
- How: "计划如何实施？"

#### 2. 内容组织层（写作阶段）

**用途**：构建内容结构，确保逻辑完整。

**故事结构应用**（Stein & Glenn, 1979 的事件地图模型）：

| 故事元素 | 对应 5W1H | 问题 |
|----------|----------|------|
| **Setting**（背景） | Who + Where + When | 谁参与？何地何时发生？ |
| **Initiating Event**（触发事件） | What + Why | 发生了什么？为什么发生？ |
| **Internal Response**（内心反应） | How | 感受如何？ |
| **Attempts**（尝试） | How | 采取了什么行动？ |
| **Consequences**（后果） | What | 结果是什么？ |
| **Resolution**（结局） | How | 最终感受如何？ |

#### 3. 质量审查层（编辑阶段）

**用途**：检查内容完整性，补充缺失要素。

**审查清单**：
- [ ] WHO：是否明确了涉及的人物/角色？
- [ ] WHAT：核心事件/内容是否清楚？
- [ ] WHEN：时间背景是否交代？
- [ ] WHERE：地点/场景是否明确？
- [ ] WHY：原因/动机是否解释？
- [ ] HOW：过程/方法是否描述？

## 详细用法

### 数据叙事中的 5W1H

**来源：Ingrid Pino, Medium (2024)**

在数据故事（Data Storytelling）中，5W1H 帮助构建完整的叙述：

| 要素 | 数据故事应用 | 示例 |
|------|-------------|------|
| **What** | 数据主题/指标 | "用户留存率" |
| **Who** | 数据主体/受众 | "新注册用户在首月" |
| **When** | 时间维度 | "2024年第一季度" |
| **Where** | 地理维度 | "北美市场" |
| **Why** | 业务背景 | "产品改版后的表现评估" |
| **How** | 数据来源/方法 | "通过 Mixpanel 追踪，样本量 10,000" |

**案例**：
> "2024年Q1（When），我们的北美新用户（Who/Where）在首月留存率（What）下降了15%（How/数据）。这发生在产品UI改版后（Why/背景），主要流失发生在 onboarding 流程的第3步（How/细节）。"

### 内容营销中的 5W1H

**来源：Ryan's Reading Notes (2023)**

应用于博客/文章策划：

| 要素 | 内容策略问题 | 应用示例 |
|------|-------------|----------|
| **Who** | 目标读者是谁？ | "初级产品经理" |
| **What** | 文章主题？ | "如何写产品需求文档" |
| **When** | 发布时间？ | "新人入职季（3月/9月）" |
| **Where** | 发布渠道？ | "LinkedIn + 公司博客" |
| **Why** | 读者为什么要读？ | "减少需求评审返工" |
| **How** | 如何组织内容？ | "PRD模板 + 案例分析 + 检查清单" |

### 项目管理中的 5W1H

**来源：Airfocus Glossary (2021)**

项目启动文档的标准框架：

```
项目名称：[What]
├─ 干系人：[Who] 涉及哪些人？利益相关者是谁？
├─ 目标：[What + Why] 要交付什么？为什么重要？
├─ 时间：[When] 开始时间？里程碑？截止日期？
├─ 地点：[Where] 执行地点？服务区域？
├─ 原因：[Why] 商业理由？预期收益？
└─ 方法：[How] 执行策略？资源需求？风险缓解？
```

### SEO 写作中的 5W1H

**来源：Ryan's Reading Notes (2023)**

应用于内容优化：

1. **关键词优化**：为每个 5W1H 要素融入相关关键词
2. **Meta Description**：用 5W1H 结构写吸引人的摘要
3. **内链策略**：每个要素链接到相关内容
4. **FAQ 页面**：直接用 5W1H 组织问题和答案

**示例**（SEO 文章大纲）：
- **What**：定义和概念解释
- **Why**：重要性和好处
- **Who**：适用人群/角色
- **When**：最佳时机/场景
- **Where**：应用场景/平台
- **How**：操作步骤/最佳实践

## 示例对比

### 示例 1：新闻报道

**场景**：报道一起商业事件

**❌ 不完整的导语**：
> "公司发布了新软件。"

**问题分析**：
- ❌ WHO：哪家公司？
- ❌ WHAT：什么软件？什么功能？
- ❌ WHEN：何时发布？
- ❌ WHERE：哪个市场？
- ❌ WHY：为什么要发布？
- ❌ HOW：如何获取？定价？

**✅ 完整的导语**（5W1H 完整）：
> "硅谷初创公司 TechFlow（Who）于本周二（When）在旧金山（Where）发布了一款AI驱动的项目管理软件（What），旨在帮助远程团队提升30%的工作效率（Why）。该软件采用订阅制，每月$29起，目前已开放公测注册（How）。"

### 示例 2：产品介绍

**场景**：介绍一款健身应用

**5W1H 框架应用**：

| 要素 | 内容 |
|------|------|
| **WHO** | "忙碌的上班族，特别是久坐的办公室人群" |
| **WHAT** | "FitDesk——一款针对办公场景的微运动应用" |
| **WHEN** | "工作时间，每90分钟提醒一次" |
| **WHERE** | "办公桌前，无需离开座位" |
| **WHY** | "久坐是健康杀手，但去健身房太耗时" |
| **HOW** | "3分钟视频指导 + 智能提醒 + 进度追踪" |

**组合成完整描述**：
> "如果你是一名每天久坐8小时的上班族（WHO），FitDesk（WHAT）能在你的办公时间（WHEN/WHERE）通过每90分钟的3分钟微运动（HOW），帮助你对抗久坐危害（WHY）——无需去健身房，就在办公桌前完成。"

### 示例 3：小说/故事构建

**来源：Writers Helping Writers / Jami Gold (2023)**

使用 5W1H 构建故事核心：

**故事问题回答**：
- **Who**：主角是谁？他们想要什么？
- **What**：核心冲突是什么？
- **When**：故事发生在什么时间？
- **Where**：故事世界/地点是怎样的？
- **Why**：为什么这个目标对主角重要？
- **How**：主角计划如何实现目标？

**示例**（简化版《火星救援》）：
- **Who**：植物学家马克·沃特尼
- **What**：在火星上独自生存并返回地球
- **When**：近未来，一次火星任务期间
- **Where**：火星表面，NASA基地
- **Why**：被误认为死亡，团队已离开，必须自救
- **How**：利用科学知识种植土豆，修复设备，与地球建立联系

### 示例 4：商业案例分析

**场景**：分析一个成功的营销案例

| 要素 | 分析内容 |
|------|----------|
| **WHO** | 执行方：Dollar Shave Club；目标受众：年轻男性 |
| **WHAT** | 发布病毒视频，推广订阅制剃须刀 |
| **WHEN** | 2012年，YouTube 崛起期 |
| **WHERE** | 主要通过 YouTube 和社交媒体传播 |
| **WHY** | 传统剃须刀太贵，订阅模式更便宜便捷 |
| **HOW** | 幽默视频（"我们的刀片太棒了"）+ 病毒传播策略 |

**分析结论**：
> "Dollar Shave Club（WHO）在2012年（WHEN）通过一支幽默的病毒视频（HOW/WHAT），在 YouTube 上（WHERE）成功挑战了吉列等传统品牌，因为它发现了年轻男性对昂贵剃须刀的不满（WHY）。"

### 创意变体

**5W2H**：添加 **How Much**（成本/程度）
- 常用于项目预算、ROI 分析

**5W1H + So What**：添加 **那又怎样？**（影响/意义）
- 常用于商业提案、影响评估

**反向 5W1H**：
- 用于虚构写作中的"红鲱鱼"技巧——故意遗漏或扭曲某个要素制造悬念

## 与其他技巧的关系

### 与 AIDA 的关系

**互补性**：
- **5W1H** 用于**内容规划**阶段，确保信息完整
- **AIDA** 用于**内容呈现**阶段，确保说服效果

**协同工作流程**：
```
5W1H（策划） → 写作 → AIDA（组织） → 4Cs（审查） → 发布
```

### 与 PAS 的关系

**5W1H 的 WHY = PAS 的 Problem 深化**：
- PAS 识别表面问题
- 5W1H 的 WHY 追问深层原因

### 与 FAB 的关系

**5W1H 的 WHAT = FAB 的 Feature**：
- 5W1H 帮助识别所有特性
- FAB 将特性转化为价值

### 与 Storytelling 的关系

**5W1H = 故事的基本骨架**：
- 所有经典故事都包含完整的 5W1H
- 缺失任何要素，故事就会显得不完整

### 与数据叙事的关系

**来源：Ingrid Pino (2024)**

5W1H 是数据故事的"叙事检查清单"：
- 确保数据讲述一个完整的故事
- 避免"只有数据点，没有故事线"

## 常见错误

### ❌ 信息过载

**问题**：试图在首段塞入所有 5W1H 答案，导致导语臃肿。

**解决**：导语回答核心 3-4 个要素，其余在正文中展开。

### ❌ 机械套用

**问题**：为了凑齐 5W1H 而添加无关紧要的信息。

**解决**：根据内容类型，灵活调整各要素的权重。

### ❌ 忽视 WHY

**问题**：详细描述 WHAT 和 HOW，但忽略了 WHY。

**解决**：WHY 通常是最重要的要素——它提供上下文和动机。

### ❌ 混淆层级

**问题**：在宏观和微观层面混用 5W1H。

**解决**：明确当前分析的层次（整体故事 vs 具体事件）。
