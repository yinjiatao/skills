# Business Casual Formula (Joel Klettke) - 写作技巧

## 1. 什么是 Business Casual Formula

**Business Casual Formula** 是由知名B2B文案撰稿人 **Joel Klettke**（Business Casual Copywriting 创始人）开发的一套SaaS高转化文案框架。该框架基于经典的问题-解决方案（Problem-Solution）结构进行扩展，通过回答6个核心问题，帮助企业在页面文案中自然引导访客完成购买决策。

与传统僵硬的B2B文案不同，Business Casual Formula强调用对话式、易懂的语气传达价值，消除"企业行话"，让复杂的技术产品变得平易近人。

## 2. 理论溯源

### 2.1 创作者背景

**Joel Klettke** 是 Business Casual Copywriting 和 Case Study Buddy 的创始人，专攻SaaS和B2B领域的转化文案。他早期从事SEO工作，后转向转化率优化（CRO）领域。Joel以其研究驱动的方法论著称，强调在写作前必须与真实客户进行访谈。

> "The biggest mistake I see in copywriting and conversion is companies who don't do their homework by interviewing their customers. Everything – your UVP, your most persuasive sales points – even your headlines – can be pulled out of conversations with real people." — Joel Klettke

### 2.2 学术根基

该框架建立在以下理论基础之上：

| 理论来源 | 核心概念 | 在框架中的体现 |
|---------|---------|--------------|
| 经典销售理论（AIDA） | 注意力→兴趣→欲望→行动 | 6个问题的递进结构 |
| 问题-解决方案模型 | 识别痛点→提供解决方案 | 框架的核心骨架 |
| 用户中心设计（UCD） | 以用户需求为中心 | "Why should I care?"放在首位 |
| 认知负荷理论 | 减少信息处理负担 | 分阶段回答核心问题 |

### 2.3 行业演化

- **2010年代初期**：SaaS行业兴起，传统软件销售文案不再适用
- **2015-2018**：Joel Klettke在服务HubSpot、Siege Media等客户过程中提炼出该框架
- **2019至今**：被广泛应用于顶级SaaS品牌的着陆页、产品页面和邮件营销

## 3. 核心逻辑：6 Questions Framework

Business Casual Formula的核心是回答以下6个问题，并根据页面位置进行策略性布局：

### 3.1 六大核心问题

| 问题 | 核心目的 | 典型位置 |
|-----|---------|---------|
| **What is it?** | 清晰定义产品/服务是什么 | 页面顶部/首屏 |
| **Who's it for?** | 明确目标受众 | 页面顶部/首屏 |
| **Why should I care?** | 传达核心价值主张（UVP） | **首屏最优先** |
| **How does it work?** | 解释产品运作机制 | 正文内容区 |
| **Why should I trust you?** | 建立信任与可信度 | 正文+社会证明区 |
| **What's next?** | 明确下一步行动 | 页面底部/CTA区 |

### 3.2 页面布局策略

根据Joel Klettke的建议，问题的回答应按以下方式分布：

```
┌─────────────────────────────────────────────────────────┐
│  【首屏区域】                                              │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │
│  1. Why should I care?  (核心价值主张 - 最优先)            │
│  2. What is it?         (产品定义)                        │
│  3. Who's it for?       (目标用户)                        │
├─────────────────────────────────────────────────────────┤
│  【正文内容区】                                            │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │
│  4. How does it work?        (产品机制)                   │
│  5. How will it improve my life? (利益转化)               │
│  6. Why should I trust you?  (信任建立)                   │
├─────────────────────────────────────────────────────────┤
│  【底部CTA区】                                             │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │
│  7. What's next?  (明确行动号召)                          │
└─────────────────────────────────────────────────────────┘
```

## 4. 详细用法：评级检查表

### 4.1 五级评估标准

使用以下检查表评估你的文案是否符合Business Casual Formula标准：

#### Ⅴ级（卓越）- 90-100分

| 检查项 | 标准 |
|-------|------|
| ☐ 首屏3秒内传达"Why should I care?" | 访客立即理解核心价值 |
| ☐ "What is it?"用一句话清晰定义 | 避免行业术语 |
| ☐ "Who's it for?"精确到角色/场景 | 使用"如果你是X，这款产品Y"格式 |
| ☐ 每个功能都连接业务成果 | "How will it improve my life?"贯穿全文 |
| ☐ 社会证明与信任信号丰富 | 包含具体数据、客户logo、案例研究 |
| ☐ CTA清晰且与内容阶段匹配 | "What's next?"无歧义 |

#### Ⅳ级（优秀）- 75-89分

| 检查项 | 标准 |
|-------|------|
| ☐ 回答了全部6个问题 | 但分布或强调不够理想 |
| ☐ 核心价值主张在首屏可见 | 但可能不够突出 |
| ☐ 产品定义清晰 | 偶有小量术语 |
| ☐ 目标受众描述较宽泛 | 未细分到具体角色 |
| ☐ 功能与利益关联存在 | 但不够系统 |
| ☐ 包含基础信任元素 | 如客户评价 |

#### Ⅲ级（合格）- 60-74分

| 检查项 | 标准 |
|-------|------|
| ☐ 回答了4-5个核心问题 | 遗漏1-2个关键问题 |
| ☐ 核心价值传达存在 | 但被埋在内容中 |
| ☐ 产品定义基本清晰 | 需要读者额外理解 |
| ☐ 目标受众暗示 | 未明确说明 |
| ☐ 功能描述为主 | 利益转化不足 |
| ☐ 信任信号薄弱 | 仅有少量评价 |

#### Ⅱ级（需改进）- 40-59分

| 检查项 | 标准 |
|-------|------|
| ☐ 仅回答2-3个核心问题 | 大部分问题未涉及 |
| ☐ 核心价值难以识别 | 首屏信息混乱 |
| ☐ 产品定义模糊 | 使用大量行业术语 |
| ☐ 目标受众不明确 | "适用于所有人"式表述 |
| ☐ 功能罗列为主 | 无利益转化 |
| ☐ 缺乏信任建立 | 无社会证明 |

#### Ⅰ级（不合格）- 0-39分

| 检查项 | 标准 |
|-------|------|
| ☐ 几乎未回答核心问题 | 内容完全公司中心 |
| ☐ 首屏仅展示公司信息 | "我们成立于X年"等 |
| ☐ 产品描述技术导向 | 充满功能和规格 |
| ☐ 无受众定位 | 未考虑读者需求 |
| ☐ 无利益转化尝试 | 纯功能清单 |
| ☐ 无CTA或CTA模糊 | "联系我们"等弱CTA |

## 5. 示例对比

### 5.1 SaaS产品页面示例

**❌ 改进前（Ⅱ级）**

> **标题**：Welcome to CloudSync Pro  
> **副标题**：The most advanced cloud synchronization platform on the market.  
> **正文**：CloudSync Pro was founded in 2015 by a team of engineers from top tech companies. Our platform uses proprietary AI algorithms to ensure maximum efficiency...  
> **CTA**：Learn More

**问题分析**：
- 首屏未回答"Why should I care?"
- 以公司历史开场，完全公司中心
- 使用"proprietary AI algorithms"等技术术语
- CTA"Learn More"无明确下一步

---

**✅ 改进后（Ⅴ级）**

> **标题**：Stop Losing Hours to File Chaos Every Week  
> **副标题**：CloudSync Pro automatically organizes and syncs your team files—so you can focus on work that matters.  
> **适用对象**：Built for marketing teams managing 1000+ assets monthly  
> **正文**：  
> **How it works**: Connect your existing storage (Google Drive, Dropbox, OneDrive). Our smart tagging system categorizes files automatically based on content.  
> **Why trust us**: 10,000+ teams use CloudSync Pro daily, including teams at Spotify and Airbnb. Average user saves 8 hours/week.  
> **CTA**：Start Free 14-Day Trial →

**改进点**：
- ✅ 首屏立即回答"Why should I care?"（Stop losing hours...）
- ✅ 清晰定义"What is it?"（自动组织和同步文件）
- ✅ 明确"Who's it for?"（月管理1000+资产的营销团队）
- ✅ 解释"How does it work?"（三步骤简化流程）
- ✅ 建立信任（10,000+用户+知名客户+具体数据）
- ✅ 明确下一步（14天免费试用）

## 7. 与其他技巧的关系

### 7.1 可组合的技巧

```
Business Casual Formula
        │
        ├── + PAS (Problem-Agitate-Solution) → 增强"Why should I care?"的情感冲击
        │
        ├── + AIDA → 在6问题框架内优化每个部分的流动
        │
        ├── + RIDBUR → 优化现有内容时使用
        │
        ├── + Data-Driven Storytelling → 强化"Why trust you"部分
        │
        └── + Hyper-Personalization → 为不同受众定制6问题答案
```

### 7.2 使用顺序建议

1. **新建页面**：Business Casual Formula（确定架构）→ PAS/AIDA（填充内容）
2. **优化现有页面**：RIDBUR（诊断问题）→ Business Casual Formula（重构架构）
3. **高价值页面**：Business Casual Formula（基础）+ Hyper-Personalization（按细分受众定制版本）

### 9.2 常见问题与解决方案

| 问题 | 解决方案 |
|-----|---------|
| "我们的产品有多个用户群" | 为每个细分受众创建独立页面或页面区块 |
| "产品很复杂，难以一句话定义" | 先传达核心价值，再将复杂解释放入"How it works" |
| "缺乏客户案例建立信任" | 使用统计数据、专家背书、方法论透明度 |
| "CTA应该放多少个" | 主CTA重复2-3次，根据滚动深度分布 |
