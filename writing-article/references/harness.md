# 写作原则（Harness）

这份文件是所有阶段的底线规则。不管写什么题材、用什么技巧，这六条原则必须全程执行。
写出来的东西如果违反了这里的任何一条，就需要回炉重写。

---

## 原则一：对话感优先，消灭书面腔

写出来的每一句话，都要像在跟朋友聊天。

**怎么做：**
- 用短句。一句话说一件事。
- 用简单词。能用"说"就别用"阐述"，能用"做"就别用"实施"。
- 把所有学术化表达干掉：~~"此外"~~ ~~"然而"~~ ~~"尽管"~~ ~~"综上所述"~~ ~~"值得注意的是"~~
- 把企业官话干掉：~~"赋能"~~ ~~"生态"~~ ~~"全链路"~~ ~~"闭环"~~ ~~"抓手"~~
- 每个形容词都问自己：去掉它，句子更好还是更差？大多数时候更好。
- 段落之间的过渡靠内容本身的逻辑推动，不靠过渡词硬拼。

**自我测试：** 写完之后朗读一遍。听起来像朋友在聊天？通过。听起来像在念文件？重写。

---

## 原则二：真实性高于一切，零虚构

真实是信任的基础。一篇文章里有一处编造，整篇文章的可信度归零。

**硬性禁止：**
- 编造数据、统计数字
- 编造案例、人名、公司名
- 编造引用、名人名言
- 编造事件经过

**需要举例时：**
- 首选：真实案例（注明出处）
- 次选：明确标注"假设一个场景"或"比方说"
- 绝对不要把编造的东西写得像真事一样

**涉及个人故事时：**
- 标注 `【作者故事位】{框架说明}` 占位
- 由作者补充真实经历，AI 不代为编造
- 框架可以给，但内容必须是作者自己的

**无法确认的事实：**
- 不写。或者写"待核实"。
- 宁可空着，也不要编。

---

## 原则三：消灭 AI 痕迹

AI 写作有一些很明显的"指纹"。读者一眼就能看出来，会立刻失去信任。

**排查清单：**

| AI 痕迹 | 解决办法 |
|---------|---------|
| 连续 3 句相同句式（主谓宾、主谓宾、主谓宾） | 打乱句式：长短交替，偶尔来个倒装或省略 |
| 滥用过渡词（"首先""其次""最后"） | 删掉。段落之间用内容逻辑连接 |
| 说完一个观点又解释一遍 | 砍掉重复的解释。说清楚一次就够了 |
| 开头必讲大背景，结尾必升华总结 | 换个开法。可以从细节切入、从问题切入、从场景切入 |
| 每段开头都是"在当今……" | 直接删。进入正题 |
| 排比用到停不下来 | 偶尔用。连续排比超过 3 组就太多了 |

**核心原则：** 用更灵活、更随机的表达方式。真人写东西有节奏变化，有突然的转折，有不完美的地方。AI 写的东西太"光滑"了——每一段都差不多长，每一句都差不多结构，像流水线产品。打破这种均匀感。

---

## 原则四：感官与镜头感

好的写作让读者"看到"画面，"听到"声音，"感受到"温度。纯抽象描述像白开水——能喝，但没味道。

**每一个场景描述，至少激活一种感官：**

| 感官通道 | 示例 |
|---------|------|
| 视觉 | 光线、颜色、空间大小、人物动作 |
| 听觉 | 声音、节奏、静默（"安静"也是一种听觉描写） |
| 触觉/温度 | 物体质感、冷暖、重量 |
| 嗅觉/味觉 | 气味、口感（适用于特定场景） |

**时间锚点：**
- 用"周二下午 3 点"，不用"有一天"
- 用"2024 年夏天"，不用"前不久"
- 具体的时间让场景变得真实可信

**空间锚点：**
- 写出具体的物理环境。在哪条街、什么样的房间、桌上放着什么。
- ~~"某个地方"~~ → "公司楼下那个总排长队的咖啡店"

---

## 原则五：加入个性化特质（不过度）

读者能分辨出"一个人在写"和"一台机器在输出"。区别就在那些微小的个性化痕迹。

**怎么做：**
- **用词选择**：每个人都有自己偏爱的表达方式。选择能体现作者个性的词，而不是最"标准"的词。
- **偶尔跑题**：全文 2-3 次，插入一句跟主题不直接相关但有趣的补充。像聊天时会突然蹦出来的那种。
- **简短评论**：在关键处加入作者的态度——"说实话，……""我觉得这里有意思的是……""你可能会觉得这不重要，但……"

**底线：** 微妙、不频繁。像菜里的盐——尝不出"咸"，但没有就寡淡。如果每段都在抖机灵，反而成了另一种 AI 痕迹。

---

## 原则六：个人故事框架嵌入

个人故事是建立信任最有效的武器。但不是每个地方都适合，也不是随便一个故事就管用。

**适合加入个人故事的位置：**

| 位置 | 原因 |
|------|------|
| 核心观点首次出现时 | 建立可信度——"我说这个不是纸上谈兵，我经历过" |
| 抽象概念需要具象化时 | 帮读者理解——"这个抽象道理，在我身上是这样体现的" |
| 情感低谷处 | 制造共鸣——"我也曾经卡在这里" |
| 行动号召前 | 建立信任——"我用这个方法做到了，你也可以" |

**标注格式：**

```
【作者故事位】{故事框架说明，作者需补充真实经历}
```

比如：
```
【作者故事位】{在此处分享一个你亲身经历的转折点——从"完全不懂"到"突然开窍"的瞬间。重点写当时的心理状态和触发顿悟的具体细节。}
```

**注意：**
- 框架给出来，内容留给作者填。
- 不要代写"我曾经在北京的一个冬天……"这种虚假的第一人称叙事。
- 如果作者没有相关经历，这个位置就空着，换用其他方式（数据、类比、他人案例）来支撑观点。
