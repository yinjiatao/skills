# 约束规则（最高优先级）

**最高优先级。与其他文件冲突时，以本文件为准。**

---

## 评级体系（软约束）

| 评级 | 含义 |
|------|------|
| Ⅴ | 强烈推荐，默认优先选择 |
| Ⅳ  | 推荐，无法满足时采用 |
| Ⅲ   | 可用，特定场景合理使用 |
| Ⅱ   | 慎用，需有明确理由 |
| Ⅰ   | 避免，主动回避 |

---

## 排版软约束

### 段落

| 评级 | 规则 |
|------|------|
| Ⅴ | 每段控制在 3～5 句，保持呼吸感 |
| Ⅴ | 长短句交替，节奏有张有弛 |
| Ⅳ  | 关键转折后单独成段 |
| Ⅱ   | 连续超过 8 句的大段落 |
| Ⅰ   | 全文段落长度高度均一 |

### 标题与分层

| 评级 | 规则 |
|------|------|
| Ⅴ | 1500 字以上的长文才使用小标题 |
| Ⅴ | 小标题不超过 12 字，简洁有力 |
| Ⅱ   | 短文（800 字以下）强行加多级标题 |
| Ⅰ   | 超过 3 层的标题嵌套 |

### 加粗

| 评级 | 规则 |
|------|------|
| Ⅴ | 全文加粗不超过 5 处，用于真正关键的词或金句 |
| Ⅱ   | 加粗比例超过全文 10% |
| Ⅰ   | 将加粗作为装饰而非强调手段 |

### Emoji

| 评级 | 规则 |
|------|------|
| Ⅴ | 朋友圈、社交文案中适度使用 |
| Ⅲ   | 公众号情感向内容中少量点缀（不超过 5 个） |
| Ⅰ   | 公众号正式内容、通用文章中大量使用 |
| Ⅰ   | 同一篇内容 emoji 密度超过每段 2 个 |

## 语言软约束

### 表达

| 评级 | 规则 |
|------|------|
| Ⅴ | 用具体意象替代抽象概念 |

| Ⅱ   | 堆砌形容词，缺乏名词和动词支撑 |
| Ⅰ   | 连续使用 3 个以上同义形容词 |

### AI 写作痕迹（硬约束）

非绝对禁用，但是必须主动回避。语义准确且无更好替代时可保留，但禁止连续堆叠。如需使用，需写作完成后使用列表展示告知用户为什么必须要使用。

#### 副词 / 形容词

| 词汇 | 评级 | 备注 |
|------|------|------|
| 深刻 | Ⅰ | 万能修饰词，不指向任何具体感受 |
| 宝贵 | Ⅰ | 价值判断空洞，读者无法验证 |
| 高效 | Ⅰ | 没有数字支撑时等于什么都没说 |
| 卓越 | Ⅰ | 自我标榜，缺乏依据 |
| 优质 | Ⅰ | 同上，广告腔 |
| 全面 | Ⅰ | 伪装成严谨，实为回避具体 |
| 系统 | Ⅰ | 常被滥用来掩盖结构混乱 |
| 深入 | Ⅰ | 与「深刻」同类，修饰语而非内容 |
| 丰富 | Ⅰ | 数量感虚假，没有列举就不成立 |
| 多元 | Ⅰ | 政策文件用语，日常写作中流于口号 |
| 扎实 | Ⅰ | 自我认证，需要读者判断，不需要作者声明 |
| 强大 | Ⅰ | 程度词滥用，与「卓越」同类 |
| 独特 | Ⅰ | 人人都说独特，说了等于没说 |
| 核心 | Ⅰ | 常作为填充词使用，去掉后句意不变 |
| 关键 | Ⅰ | 同上，去掉「关键」句子照样成立 |

#### 动词短语

| 词汇 | 评级 | 备注 |
|------|------|------|
| 值得关注 | Ⅰ | 没有说为什么值得，等于没说 |
| 不可忽视 | Ⅰ | 警告腔，缺乏论据支撑 |
| 至关重要 | Ⅰ | 程度最高但最无信息量 |
| 发挥重要作用 | Ⅰ | 官方文件惯用句，意义模糊 |
| 取得显著成效 | Ⅰ | 成效是什么？显著体现在哪里？ |
| 赋能 | Ⅰ | 企业黑话，指向不明 |
| 助力 | Ⅰ | 同上，公关稿标配 |
| 推动……发展 | Ⅰ | 无主语、无机制、无结果，三无句式 |
| 带来……体验 | Ⅰ | 体验是什么体验？需要具体描述 |
| 持续探索 | Ⅰ | 行动虚化，没有说探索什么、怎么探索 |
| 不断优化 | Ⅰ | 同上，过程词掩盖了结果的缺席 |
| 积极应对 | Ⅰ | 态度词代替了行动描述 |

#### 结构词

| 词汇 | 评级 | 备注 |
|------|------|------|
| 首先……其次……最后 | Ⅰ | 机械列举，掩盖了逻辑关系的缺失 |
| 总而言之 | Ⅰ | 多余的收尾信号，直接写结论即可 |
| 综上所述 | Ⅰ | 同上，学生作文标配 |
| 不言而喻 | Ⅰ | 既然不言而喻就不要写，写了说明心虚 |
| 与此同时 | Ⅰ | 过渡词滥用，多数情况直接删掉更流畅 |
| 值得一提的是 | Ⅰ | 卖关子前摇，去掉后直接说反而更有力 |
| 毋庸置疑 | Ⅰ | 与「不言而喻」同类，强调反而显心虚 |
| 一方面……另一方面 | Ⅰ | 不是在呈现张力，是在机械对称 |
| 由此可见 | Ⅰ | 推导词，但多数情况下推导关系并不成立 |

#### 开篇句式

| 词汇 | 评级 | 备注 |
|------|------|------|
| 在当今社会…… | Ⅰ | 用宏观背景拖延进入正题 |
| 随着时代的发展…… | Ⅰ | 永远正确等于没有立场 |
| 在快节奏的现代生活中…… | Ⅰ | 套话开场，读者会直接跳过 |
| 众所周知…… | Ⅰ | 用共识感规避自己提出论点的责任 |
| 自古以来…… | Ⅰ | 历史背书开场，与主题通常无实质关联 |
| 近年来，越来越多的人…… | Ⅰ | 无数据支撑的趋势描述，读者无法核实 |
| 作为……，我们需要…… | Ⅰ | 用身份绑架读者，强行制造「我们」 |
| 你是否曾经……？ | Ⅰ | 反问开场已成模板，失去制造共鸣的效果 |

## 绝对禁止规则 ⛔（硬约束）

### 内容禁区

| # | 禁止内容 |
|---|---------|
| H-01 | 违反发布平台法律法规的内容 |
| H-02 | 涉及政治敏感人物或事件的攻击性内容 |
| H-03 | 色情、低俗内容 |
| H-04 | 真实人物的虚假言论或捏造事件 |
| H-05 | 基于性别、种族、地域、职业的歧视性内容 |
| H-06 | 对具体竞品的恶意贬低 |
| H-07 | 连续复制他人原文超过 10 字（引用除外） |

### 排版禁区

| # | 禁止行为 |
|---|---------|
| F-01 | 社交文案（social 类型）中使用任何 Markdown 标记 |
| F-02 | 正文出现 `[此处补充]` 等占位符而未填写 |
| F-03 | 同一内容在不同段落重复表达超过 2 次 |
