## 1 · 核心骨架（Architecture）

### 完整 12 阶段模型（Campbell → Vogler）

Joseph Campbell 在 1949 年的《千面英雄》里发现了一件事：全世界的英雄故事，不管是希腊神话、佛陀成道还是孙悟空取经，骨架几乎一模一样。Christopher Vogler 把 Campbell 的 17 阶段压缩到 12 个，变成了好莱坞的"故事操作手册"。

三幕十二阶段：

```
┌─────────────────────────────────────────────────────────┐
│                    ACT I · 启程                          │
│  ① Ordinary World        日常世界                        │
│  ② Call to Adventure     冒险召唤                        │
│  ③ Refusal of the Call   拒绝召唤                        │
│  ④ Meeting the Mentor    遇见导师                        │
│  ⑤ Crossing the Threshold 跨越门槛                       │
├─────────────────────────────────────────────────────────┤
│                    ACT II · 历练                         │
│  ⑥ Tests, Allies, Enemies 考验·盟友·敌人                 │
│  ⑦ Approach to the Cave   逼近深渊                       │
│  ⑧ The Ordeal             生死考验                       │
│  ⑨ The Reward             获得奖赏                       │
├─────────────────────────────────────────────────────────┤
│                    ACT III · 归来                        │
│  ⑩ The Road Back          踏上归途                       │
│  ⑪ The Resurrection       终极蜕变                       │
│  ⑫ Return with the Elixir 携灵药归来                     │
└─────────────────────────────────────────────────────────┘
```

每个阶段的核心功能：

| # | 阶段 | 一句话功能 | 读者感受 |
|---|------|-----------|---------|
| ① | 日常世界 | 建立"旧的我"作为对比基线 | 这个人跟我差不多 |
| ② | 冒险召唤 | 打破平衡——一个事件、一句话、一个念头 | 要出事了 |
| ③ | 拒绝召唤 | 展示恐惧和犹豫，赌注变得真实 | 换我也会怕 |
| ④ | 遇见导师 | 获得工具、信念或一句点醒的话 | 有人帮他了 |
| ⑤ | 跨越门槛 | 不可逆地离开舒适区 | 回不去了 |
| ⑥ | 考验·盟友·敌人 | 在新世界里摸爬滚打，学新规则 | 这条路真不好走 |
| ⑦ | 逼近深渊 | 靠近最大的恐惧，紧张感拉满 | 心开始揪起来 |
| ⑧ | 生死考验 | 直面最深的恐惧——象征性的"死亡" | 他撑不住了吧？ |
| ⑨ | 获得奖赏 | 活下来了，拿到了某种"宝物" | 松了口气 |
| ⑩ | 踏上归途 | 把收获带回原来的世界 | 事情还没完 |
| ⑪ | 终极蜕变 | 最后的公开考验，证明自己真正变了 | 他真的不一样了 |
| ⑫ | 携灵药归来 | 把收获分享给他人，完成循环 | 值了 |

### 简化 5 阶段版本（短内容适用）

12 阶段对于公众号文章、演讲、短视频来说太重了。压缩到 5 步，骨架不变，密度更高：

```
平凡世界 ──→ 召唤 ──→ 考验 ──→ 蜕变 ──→ 归来
  "旧我"     打破     受苦     觉醒     "新我"
```

| 简化阶段 | 对应完整阶段 | 核心动作 | 时间占比建议 |
|---------|------------|---------|------------|
| 平凡世界 | ①②③ | 展示旧状态 + 触发事件 + 内心挣扎 | 15-20% |
| 召唤 | ④⑤ | 获得启发 + 迈出第一步 | 10-15% |
| 考验 | ⑥⑦⑧ | 碰壁、失败、至暗时刻 | 30-40% |
| 蜕变 | ⑨⑩⑪ | 顿悟、新认知、证明自己 | 15-20% |
| 归来 | ⑫ | 分享收获，回馈读者 | 10-15% |

### 认知循环（Cognitive Loop）

英雄之旅不只是"一条线往前走"。每个阶段都包含一个微型循环：

```
感知（我碰到了什么）
  → 抗拒（我不想面对）
    → 行动（硬着头皮上）
      → 整合（我从中学到了什么）
        → 进入下一个阶段……
```

这意味着：不是只有"⑧ 生死考验"才有冲突。每个阶段都自带一轮小的"抗拒→突破"。写的时候如果每个阶段都铺一层这个微循环，故事的质感会密很多。

## 2 · 认知机理（Cognitive Mechanism）

### 为什么英雄之旅"管用"

不是因为 Campbell 厉害，是因为人类大脑本来就这么处理经历。

**原型共鸣（Archetypal Resonance）**

Carl Jung 的"集体无意识"理论说，人类共享一套深层心理结构。英雄、导师、阴影、门槛——这些不是文学概念，是大脑里预装的认知模板。读者不需要学过神话学就能"感觉到"这个结构。5 岁小孩看《狮子王》会哭，不是因为他懂三幕式，是因为他的大脑本能地响应了这个模式。

**认知失调与再平衡（Cognitive Dissonance → Resolution）**

英雄之旅的底层是一个认知失调循环：

| 阶段 | 认知状态 | 大脑反应 |
|------|---------|---------|
| 日常世界 | 平衡（但有缺失） | 低唤醒，建立基线 |
| 召唤 + 拒绝 | 失调——旧认知无法处理新信息 | 紧张感上升，注意力锁定 |
| 考验 | 持续失调——旧方法不断失败 | 大脑在搜索新模式 |
| 蜕变 | 重新平衡——新认知替代旧认知 | 多巴胺释放，"啊哈"体验 |
| 归来 | 新平衡 | 满足感、完成感 |

**叙事传输效应（Narrative Transportation）**

心理学家 Melanie Green 和 Timothy Brock 的研究发现：当人沉浸在一个故事中时，大脑的批判性思维会显著降低。英雄之旅特别擅长制造这种"传输"效果，因为它的节奏跟人类处理经历的自然节奏完全吻合——读者不是在"看别人的故事"，是在"重新经历自己的故事"。

**身份投射（Identity Projection）**

读者会把自己投射到英雄身上。这不是比喻——fMRI 研究显示，人在阅读角色经历痛苦时，大脑的痛觉相关区域真的会被激活。英雄之旅的 12 阶段精确覆盖了人类面对重大转变的心理全过程：否认→恐惧→挣扎→顿悟→重生。读者跟着走完这个过程，等于在安全环境里完成了一次心理排练。

### 记忆编码优势

故事结构的记忆留存率远高于纯信息。认知心理学给出的数据：

- 纯事实列表 → 7 天后记住约 10%
- 同样信息嵌入英雄之旅结构 → 7 天后记住约 65%

原因：英雄之旅激活了多个记忆编码通道——情节记忆（事件顺序）、情感记忆（情绪起伏）、具身记忆（感官细节）同时工作。三个通道互相加固。

## 3 · 情感引擎（Affective Engine）

### 情感拓扑图（Emotional Topology）

英雄之旅不是一条平滑的上升线。它是一条有明确高低起伏的情感地形图：

```
情感强度
  ▲
  │          ⑧生死考验
  │            ╱╲
  │           ╱  ╲    ⑪终极蜕变
  │     ⑦逼近╱    ╲     ╱╲
  │        ╱      ╲   ╱  ╲___⑫归来
  │  ⑤门槛╱   ⑥考验 ╲⑨╱ ⑩归途
  │      ╱             ╲╱
  │ ②召唤╱
  │   ╱
  │①╱ ③拒绝（情感低谷1）
  │╱
  ┼──────────────────────────────→ 时间
```

关键情感节点：

| 节点 | 情感类型 | 强度 | 写作重点 |
|------|---------|------|---------|
| ① 日常世界 | 平淡中的不满足 | ★★☆☆☆ | 让读者感到"熟悉的不舒服" |
| ③ 拒绝召唤 | 恐惧、自我怀疑 | ★★★☆☆ | 第一个情感低谷——制造共情 |
| ⑤ 跨越门槛 | 决心混着恐惧 | ★★★☆☆ | "明知山有虎"的勇气瞬间 |
| ⑧ 生死考验 | 绝望、崩溃 | ★★★★★ | 全文情感最低点——黑暗之夜 |
| ⑨ 获得奖赏 | 释放、喜悦 | ★★★★☆ | 第一波情感释放——喘息 |
| ⑪ 终极蜕变 | 坚定、超越 | ★★★★★ | 全文情感最高点——证明自己 |
| ⑫ 归来 | 平静中的充实 | ★★★☆☆ | 温暖的闭合感 |

### 情感间距原则

两个情感极端之间的落差越大，读者的体验越强烈。这是英雄之旅最厉害的地方——它不是单纯往上走，而是先把你摔到谷底（⑧），再把你拉到山顶（⑪）。

操作要点：

- **⑧ 之前必须有充分的铺垫**。如果考验阶段写得太轻松，生死考验的冲击力会大打折扣。就像过山车——爬坡越慢越长，俯冲时肾上腺素越猛。
- **⑨ 不能太快乐**。拿到奖赏之后应该有隐忧，读者心里要留一个"但事情真的结束了吗？"的疑问。
- **⑫ 的情感温度要比 ① 高，但不需要爆炸性高**。归来是温暖的完成感，不是狂欢。像长跑冲线后大口呼吸的那几秒——不兴奋，但很满。

### 镜像神经元调用（Mirror Neuron Engagement）

镜像神经元让人看到别人的动作和表情时，自己大脑里对应的区域也会激活。写英雄之旅时有意识地利用这个机制：

**身体化描写——让读者的身体跟着英雄反应：**

| 阶段 | 身体化写法示例 | 激活的感官 |
|------|--------------|-----------|
| ③ 拒绝 | "胃像被攥了一下" / "后背一层冷汗" | 内脏感觉 |
| ⑤ 门槛 | "深吸一口气，指尖碰到门把手" | 触觉+呼吸 |
| ⑧ 考验 | "膝盖撑不住了，跪在地上" / "眼前一片模糊" | 本体感觉+视觉 |
| ⑨ 奖赏 | "肩膀终于松下来" / "嘴角不受控制地往上抬" | 肌肉放松 |
| ⑪ 蜕变 | "站直了，脊椎一节一节撑起来" | 姿态变化 |

**对话引爆情感——在关键节点用一句短对话替代大段描写：**

- ④ 导师的话："你以为你需要的是答案，其实你需要的是出发。"
- ⑧ 绝望时刻："我做不到"——三个字比三段描写有力。
- ⑪ 蜕变时刻："这次轮到我来。"——身份转换的标志。

### 情感过渡不要跳跃

最常见的失败：从 ⑧ 直接跳到 ⑪，中间的 ⑨⑩ 草草带过。读者的情感需要台阶式地回升，不能坐电梯。⑧ 是地下室，⑪ 是天台——你得一层一层爬。每一层都需要一个小的情感事件来"垫"。

## 4 · 视觉修辞（Visual Rhetoric）

### 心智画面（Mental Imagery）

英雄之旅的每个阶段都应该有一个"标志性画面"——读者闭上眼能看到的那种。

| 阶段 | 典型视觉意象 | 示例 |
|------|-------------|------|
| ① 日常世界 | 重复、封闭、灰色调 | 每天早上同一个地铁口，同一杯美式，同一张工位 |
| ② 召唤 | 裂缝、光线、突然出现的东西 | 一封邮件、一个电话、一个意外的相遇 |
| ③ 拒绝 | 背对、关门、缩小 | 把手机扣在桌上、关掉浏览器标签页 |
| ④ 导师 | 递过来的东西——一本书、一句话、一个眼神 | 咖啡馆里，对面的人推过来一张纸 |
| ⑤ 门槛 | 跨过、穿过、离开 | 提着行李站在门口，回头看了一眼空荡荡的房间 |
| ⑥ 考验 | 混乱、碎片、快速切换 | 无数个深夜、无数次失败的截图、堆满便利贴的墙 |
| ⑦ 逼近 | 黑暗、狭窄、呼吸变浅 | 走廊越来越暗，空气越来越稠 |
| ⑧ 生死考验 | 坠落、溺水、碎裂 | 地板从脚下消失的那一秒 |
| ⑨ 奖赏 | 光线回来、手里多了什么 | 黎明、一封回信、一个拥抱 |
| ⑩ 归途 | 路、方向、速度 | 从机场出来，空气跟记忆里的不一样了 |
| ⑪ 蜕变 | 站起来、面对、眼神变了 | 镜子里的自己，眼睛跟三个月前不一样 |
| ⑫ 归来 | 圆、循环、交接 | 把自己经历的东西递给下一个人 |

### 镜头运动表（Cinematic Movement Table）

把文字当镜头。不同阶段用不同的"镜头语言"：

| 阶段 | 镜头类型 | 节奏 | 句子特征 |
|------|---------|------|---------|
| ① 日常世界 | 固定机位·中景 | 慢 | 平铺直叙，句子长度均匀 |
| ②③ 召唤+拒绝 | 推进·特写 | 渐快 | 句子开始变短，出现省略 |
| ④ 导师 | 双人镜头·正反打 | 稳 | 对话为主，留白多 |
| ⑤ 门槛 | 跟拍·移动 | 一个节拍的停顿后加速 | 一个长句+一个极短句 |
| ⑥ 考验 | 快切·蒙太奇 | 快 | 短句连发，场景跳切 |
| ⑦ 逼近 | 缓慢推进·浅焦 | 极慢 | 细节描写密度最高 |
| ⑧ 生死考验 | 手持·晃动·失焦 | 先极快后骤停 | 碎片化句子→突然安静→一个长句 |
| ⑨ 奖赏 | 拉远·全景 | 舒缓 | 句子重新变长，出现环境描写 |
| ⑩ 归途 | 跟拍·稳定器 | 匀速 | 叙述感回归，带着方向感 |
| ⑪ 终极蜕变 | 升格（慢动作）·大特写 | 时间拉长 | 每个动作拆开写，一帧一帧 |
| ⑫ 归来 | 固定机位·远景 | 渐慢 | 回归 ① 的句式，但温度不同 |

### 场景锚定（Scene Anchoring）

每个阶段绑定一个具体的物理空间。不要在"抽象空间"里讲故事。

操作方法：

1. **选一个"主场景"作为日常世界的视觉锚**——比如"工位"、"出租屋"、"那条上班必经的路"。
2. **门槛场景必须是物理的跨越**——不是"我决定了"，是"我走出了那扇门"。
3. **生死考验有一个封闭空间**——酒店房间、深夜的办公室、一个人的出租车后座。
4. **归来场景要跟日常世界在同一个物理空间**——但有一个细节变了。同一条路，但你看到了之前没注意的东西。

这种"空间回环"让读者在视觉层面也感受到"英雄变了"。不用说"我成长了"——同一个地铁口，他的姿势不一样了，读者自己会明白。

## 5 · 执行清单（Execution Protocol）

写作时逐阶段对照。每个阶段完成后打勾，再进入下一个。

### 完整 12 阶段清单

**ACT I · 启程**

**① 日常世界**
- [ ] 用一个具体场景（不是概述）展示英雄的日常
- [ ] 让读者在 3 句话内感到"这个人跟我有点像"
- [ ] 埋下一个"缺失"——英雄生活里缺了什么，他自己可能还没意识到
- [ ] 至少激活两种感官通道
- [ ] 不要写"从前有一个人"——直接把读者扔进场景

**② 冒险召唤**
- [ ] 用一个具体事件打破平衡（不是"慢慢意识到"）
- [ ] 事件必须足够有力——读完这一段，读者知道"旧生活回不去了"
- [ ] 召唤的形式要跟英雄的世界匹配（程序员不会收到羊皮卷）

**③ 拒绝召唤**
- [ ] 展示真实的恐惧（不是装腔作势的犹豫）
- [ ] 让读者感到"换我也会怕"——恐惧要具体，不要抽象
- [ ] 可以很短（一两段就够），但不能跳过
- [ ] 拒绝的理由要合理——责任、恐惧、自我怀疑，不是无理取闹

**④ 遇见导师**
- [ ] 导师不一定是人——可以是一本书、一个视频、一段经历
- [ ] 导师提供的不是"答案"，是"工具"或"视角"
- [ ] 导师场景要有具体的对话或互动（不要"他告诉我很多道理"）
- [ ] 【作者故事位】如果是非虚构，这里标注让作者填真实的"导师时刻"

**⑤ 跨越门槛**
- [ ] 有一个物理动作标记跨越（离开、登机、签字、按下发送键）
- [ ] 这个决定是不可逆的——烧掉退路
- [ ] 用一个短句收束这个阶段："就这样，我出发了。"

**ACT II · 历练**

**⑥ 考验·盟友·敌人**
- [ ] 至少写 2-3 个具体的考验事件（不要"经过了很多困难"）
- [ ] 每个考验教会英雄一样东西
- [ ] 盟友和敌人要有名字和面孔（不要"一个朋友告诉我"）
- [ ] 节奏用蒙太奇——快速切换，不在单个事件上停留太久

**⑦ 逼近深渊**
- [ ] 紧张感持续上升——用越来越短的句子
- [ ] 英雄知道接下来要面对什么，但还是往前走
- [ ] 环境描写变暗、变窄、变安静
- [ ] 这是"暴风雨前的宁静"——可以有一段反常的平静

**⑧ 生死考验**
- [ ] 这是全文情感最低点——英雄必须"死"一次（象征性的）
- [ ] 旧的自我认知在这里彻底碎裂
- [ ] 不要着急"救"英雄——让他在谷底多待一会儿
- [ ] 用碎片化的句子、感官细节、身体反应来写——不要用"我很绝望"
- [ ] 【作者故事位】非虚构写作中，这里是最需要真实经历的地方

**⑨ 获得奖赏**
- [ ] 英雄从谷底得到了什么——一个新的认知、一种新的能力、一段关系的修复
- [ ] 奖赏要具体——不是"我变强了"，是"我终于明白了 XXX"
- [ ] 留一丝隐忧——事情没有完全结束

**ACT III · 归来**

**⑩ 踏上归途**
- [ ] 英雄开始往回走，把收获带回原来的世界
- [ ] 可以很短——一个过渡段落就够
- [ ] 如果有最后一个障碍，在这里设置

**⑪ 终极蜕变**
- [ ] 英雄在公开场合证明自己真的变了
- [ ] 这是全文情感最高点——用升格镜头（慢动作写法）
- [ ] 跟 ⑧ 形成对照——同样的困境，不同的反应
- [ ] 用行动展示变化，不要用"我终于明白了"这种直白陈述

**⑫ 携灵药归来**
- [ ] 英雄的收获惠及他人——不只是自己变好了
- [ ] 回到日常世界的场景，但有一个细节变了
- [ ] 最后一段回应开头——形成闭环
- [ ] 如果面向读者：明确告诉读者"你也可以"

### 简化 5 阶段清单（短内容用）

**平凡世界（15-20%）**
- [ ] 一个场景展示"旧的状态"
- [ ] 一个事件打破平衡
- [ ] 一句话的内心挣扎

**召唤（10-15%）**
- [ ] 点明推动力——是什么让你迈出第一步
- [ ] 一个物理动作标记"出发"

**考验（30-40%）**
- [ ] 至少两个具体困难
- [ ] 一个"至暗时刻"——不要跳过这个
- [ ] 至暗时刻要有身体感受的描写

**蜕变（15-20%）**
- [ ] 一个清晰的"啊哈"时刻
- [ ] 用行动（不是感悟）展示变化

**归来（10-15%）**
- [ ] 把收获翻译成读者能带走的东西
- [ ] 回应开头的场景或意象

## 6 · 场景适配（Contextual Matrix）

不同平台和内容形态下，英雄之旅的使用方式差别很大。

### 平台适配表

| 平台 | 推荐版本 | 字数 | 重点阶段 | 注意事项 |
|------|---------|------|---------|---------|
| 公众号长文 | 完整 12 阶段 | 3000-8000 | 全阶段展开 | 每 800 字一个节奏变换点，避免"从头到尾一样重" |
| 公众号短文 | 简化 5 阶段 | 1500-2500 | 考验 + 蜕变 | 日常世界可以压到 2-3 句，快速进入召唤 |
| 知乎回答 | 简化 5 阶段 | 1000-3000 | 平凡世界→考验（用来建立可信度） | 知乎读者吃"真实感"——考验部分要有细节 |
| 小红书 | 极简 3 步：旧我→转折→新我 | 300-800 | 蜕变 | 前后对比拉满，视觉化呈现（图片辅助） |
| 即刻 | 简化 5 阶段（压缩版） | 500-1500 | 考验 + 蜕变 | 口语化，像在跟朋友讲一件事 |
| LinkedIn | 简化 5 阶段 | 800-2000 | 召唤 + 蜕变 + 归来 | "归来"部分要有行业洞见，不能只讲个人 |
| Medium | 完整 12 阶段 | 2000-5000 | 全阶段展开 | 英文写作可以更直接，不用太多铺垫 |
| B 站视频 | 简化 5 阶段 | 脚本 1500-3000 | 考验（画面最多） | 用画面和剪辑辅助情感节奏 |
| 抖音/短视频 | 极简 3 步 | 脚本 200-500 | 蜕变瞬间 | 前 3 秒必须是最有冲击力的画面——倒叙开头 |
| 演讲/TED | 完整 12 阶段（口语化） | 口播 2000-4000 | ⑧ 生死考验（演讲的高潮） | 导师阶段可以变成"致谢"，归来阶段变成 Call to Action |
| 播客 | 完整 12 阶段（对话版） | 对话 5000-10000 | ⑥ 考验（最多细节） | 主持人扮演"读者代理"，在关键节点追问 |

### 内容类型适配

| 内容类型 | 英雄是谁 | 侧重点 | 常见陷阱 |
|---------|---------|--------|---------|
| 个人成长文 | 作者自己 | ⑧ 生死考验要真实——编的东西读者一眼看穿 | 把自己写得太完美，没有真正的"低谷" |
| 品牌故事 | 创始人 / 品牌 | ② 召唤要有社会意义——不只是"想赚钱" | 把困难写得太轻，像凡尔赛 |
| 客户案例 | 客户 | ① 日常世界要让目标受众共情 | 产品介入太早——应该在 ④ 导师阶段自然出现 |
| 知乎盐选 | 虚构/真实主角 | ⑥⑦⑧ 三个阶段要浓墨重彩 | 蜕变来得太突然，缺少铺垫 |
| TED 演讲 | 演讲者 | ⑫ 归来要转化成给听众的行动指令 | 在考验阶段停留太久，没时间展示蜕变 |
| 小说 | 虚构主角 | 内在旅程和外在旅程双线并行 | 只有外在事件，没有心理变化 |

## 7 · 组合策略（Technique Stacking）

英雄之旅是一个"宏观骨架"，需要跟其他技巧配合才能出细活。

### 推荐组合

| 组合 | 搭配技巧 | 怎么配合 | 适用场景 |
|------|---------|---------|---------|
| 骨架 + 说服 | Hero's Journey + APAG | 英雄之旅提供叙事线，APAG 的 Promise-Argue-Guide 嵌入 ⑫ 归来阶段 | 个人成长 + 行动号召 |
| 骨架 + 开头 | Hero's Journey + PAS | 用 PAS 的 Pain-Agitate-Solve 来写 ①②③（日常→召唤→拒绝） | 需要强开头的内容 |
| 骨架 + 产品 | Hero's Journey + Before-After-Bridge | 整个英雄之旅就是一个放大版的 Before-After，Bridge 是产品/方法 | 客户案例、品牌故事 |
| 骨架 + 细节 | Hero's Journey + Show Don't Tell | 在 ⑦⑧⑪ 三个关键阶段用 Show Don't Tell 替代直白描述 | 需要沉浸感的叙事 |
| 骨架 + 节奏 | Hero's Journey + Energy Transfer | 用 Energy Transfer 的对话感和节奏控制贯穿全文 | 所有使用英雄之旅的场景 |
| 骨架 + 记忆 | Hero's Journey + Rule of 3 | 考验阶段安排 3 个递进的困难；归来阶段给 3 个收获 | 演讲、长文 |
| 骨架 + 品牌 | Hero's Journey + StoryBrand | StoryBrand 的 7 步正好映射到英雄之旅（客户=英雄，品牌=导师） | 品牌叙事、营销内容 |

### 阶段内嵌技巧速查

| 英雄之旅阶段 | 可嵌入的技巧 | 嵌入方式 |
|-------------|------------|---------|
| ①② 日常世界 + 召唤 | PAS 的 Pain + Agitate | 日常世界=Pain，召唤=Agitate |
| ④ 导师 | Golden Circle (Why-How-What) | 导师的话用 Golden Circle 结构组织 |
| ⑥ 考验 | Rule of 3 | 三个递进考验，每个比上一个难 |
| ⑧ 生死考验 | Show Don't Tell | 不说"我崩溃了"，展示崩溃的细节 |
| ⑨⑪ 奖赏 + 蜕变 | Before-After-Bridge | 蜕变前后的对比 |
| ⑫ 归来 | APAG 的 Guide | 把个人收获翻译成读者的行动指南 |

### 反面组合（不推荐）

| 组合 | 为什么不行 |
|------|-----------|
| Hero's Journey + AIDA | AIDA 是转化漏斗，逻辑跟叙事弧线冲突——硬嫁接会让故事变成推销 |
| Hero's Journey + SCQA（完整版） | SCQA 的 Situation-Complication 跟英雄之旅的 ①②③ 功能重叠，用一个就够 |
| Hero's Journey + FABE | FABE 是产品特性说明框架，跟叙事不兼容——除非产品出现在 ④ 导师位 |

## 8 · 反向工程（Reverse Engineering）

拿到一篇好的英雄之旅文章或演讲，按以下步骤拆解。

### 解构协议（Deconstruction Protocol）

**第一步：标记阶段**

读完全文，用不同颜色标记每个阶段的起止位置。先标 12 阶段版本，如果不是所有阶段都在，再试 5 阶段版本。

标记格式：

```
[①日常] 第 1-3 段
[②召唤] 第 4 段
[③拒绝] 第 5 段第 2 句 - 第 6 段
...
```

**第二步：提取情感曲线**

为每个阶段打一个情感分数（1-10），画出实际的情感曲线，然后跟"标准拓扑图"（见第 3 节）对比：

- 哪里偏高了？——可能是作者刻意制造的意外
- 哪里偏低了？——可能是薄弱环节
- 有没有"平台期"？——连续 2-3 个阶段情感强度差不多，节奏会拖

**第三步：检查三个关键比**

| 比值 | 理想范围 | 说明 |
|------|---------|------|
| ACT I : ACT II : ACT III | 25% : 50% : 25% | 经典三幕比例。ACT II 太短=考验不够，太长=拖沓 |
| ⑧ 低谷 : ⑪ 高峰 情感落差 | ≥ 5 分 | 落差越大，读者体验越强 |
| 内在旅程 : 外在旅程 | 至少 40:60 | 纯外在事件没有灵魂，纯内在思考没有画面 |

**第四步：找"金句"和"锚点场景"**

标出全文里你记住的句子和画面。好的英雄之旅里，每个阶段至少有一个让你记住的东西。如果某个阶段你完全没印象——那就是写得不够好的地方。

### 重建练习（Reconstruction Drills）

用以下练习来训练英雄之旅的"肌肉记忆"。

**练习 1：阶段打乱重组**

拿一篇你熟悉的英雄之旅故事，把 12 个阶段打乱顺序，重新排列。哪些阶段可以调换位置？哪些绝对不能？

- 可调换的：⑥ 和 ⑦ 之间可以互换、穿插
- 绝对不能：⑧ 必须在 ⑨ 之前，⑪ 必须在 ⑫ 之前

这个练习帮你区分"可选的节奏调整"和"不可违反的因果逻辑"。

**练习 2：同一个故事，三种长度**

选一个你自己的经历，分别写成：

| 版本 | 字数 | 用哪个版本 | 重点 |
|------|------|-----------|------|
| 抖音脚本 | 200 字 | 极简 3 步（旧我→转折→新我） | 只保留 ⑧ 和 ⑪ |
| 公众号短文 | 1500 字 | 简化 5 阶段 | 考验部分拉开写 |
| 深度长文 | 4000 字 | 完整 12 阶段 | 每个阶段都有具体场景 |

对比三个版本——哪些细节在压缩时必须砍掉？哪些砍掉之后故事就不成立了？后者就是你故事的核心骨架。

**练习 3：角色互换**

同一个故事，换一个"英雄"重新讲：

- 客户案例：从客户视角换成客户的同事视角
- 个人成长：从自己视角换成身边人的视角
- 品牌故事：从创始人视角换成第一个用户的视角

每换一次视角，12 个阶段的具体内容都会变——但骨架不变。这个练习帮你理解"结构是固定的，填充是灵活的"。

**练习 4：缺陷阶段修复**

找一篇你觉得"差了点什么"的文章，用 12 阶段模型诊断，找出缺失或薄弱的阶段，单独重写那个阶段。常见问题：

| 症状 | 可能缺失的阶段 |
|------|--------------|
| 开头没有代入感 | ① 日常世界写得太抽象 |
| 转变来得太突然 | ⑥⑦ 考验和逼近太短 |
| 读完没有触动 | ⑧ 生死考验不够深 |
| 结尾空洞 | ⑫ 归来没有给出"灵药" |
| 整篇平平淡淡 | 情感曲线没有起伏——所有阶段在同一个情感强度 |
