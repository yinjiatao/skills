# 通用（无特定平台）

## 适用场景

用户未指定发布平台，或内容用于内部流通时加载。

---

## 字数规范

| 类型 | 建议字数 |
|------|---------|
| 短文 | 300～800 字 |
| 常规文章 | 800～2000 字 |
| 深度长文 | 2000～5000 字 |

---

## 排版规范

| 评级 | 规则 |
|------|------|
| Ⅴ | 使用 Markdown，段落间空行，保持视觉呼吸感 |
| Ⅴ | 每段控制在 4～6 行，避免大段文字堆砌 |
| Ⅴ | 1500 字以上使用 `##` / `###` 小标题分层 |
| Ⅳ | 列表项保持并行结构，同一列表风格统一 |
| Ⅳ | 关键信息可加粗，每段不超过 1 处 |
| Ⅱ | 连续超过 3 个二级标题而无正文过渡（文章骨架化） |
| Ⅱ | 无实质内容的过渡句（"接下来我们来看……"） |
| Ⅰ | 滥用标题层级（每段都加标题） |
| Ⅰ | 大量使用粗体导致重点失焦 |

---

## 标题写法

| 评级 | 规则 |
|------|------|
| Ⅴ | 直接点明读者收益或核心问题 |
| Ⅴ | 控制在 20 字以内，信息密度高 |
| Ⅳ | 设悬念 / 提问题 / 反常识，引发好奇 |
| Ⅳ | 含具体数字或场景，增强可信度 |
| Ⅱ | 过度修辞导致含义模糊 |
| Ⅰ | 标题与正文内容脱节 |
| Ⅰ | 纯感叹式标题，无信息量（如"太厉害了！"） |

---

## 语气与风格

| 评级 | 规则 |
|------|------|
| Ⅴ | 与目标读者的知识背景匹配，不降维也不过度专业 |
| Ⅴ | 立场一致，全文语气统一 |
| Ⅳ | 长句与短句交替，避免节奏单调 |
| Ⅱ | 过度使用被动语态（"被认为是""可以被看作"） |
| Ⅱ | 堆砌形容词，缺乏实质支撑 |
| Ⅰ | 口号式表达，空洞无内容（"赋能""颠覆式创新"） |

---

## 结构与逻辑

| 评级 | 规则 |
|------|------|
| Ⅴ | 开篇点明主题或读者收益，不绕弯子 |
| Ⅴ | 段落内部：论点 → 论据 → 小结，逻辑自洽 |
| Ⅴ | 全文有清晰的起承转合，结尾呼应开头或给出行动建议 |
| Ⅳ | 复杂论点先给结论，再展开细节（结论前置） |
| Ⅱ | 论点与论据之间缺乏显式连接 |
| Ⅰ | 大量铺垫之后才出现核心观点（读者已流失） |

---

## 内容质量

| 评级 | 规则 |
|------|------|
| Ⅴ | 每段只承载一个核心信息，删除可去掉的句子后意思不变则删 |
| Ⅴ | 用具体例子、数据或场景支撑抽象论点 |
| Ⅳ | 主动说明内容的局限性或适用边界 |
| Ⅱ | 凑字数的重复表述（同一意思换词再说一遍） |
| Ⅱ | 引用数据不注明来源或时效 |
| Ⅰ | AI 味明显：泛泛而谈、没有立场、结构过于整齐对称 |
| Ⅰ | 开头套话（"在当今社会……""随着时代的发展……"） |

---

## 受众假设

无特定受众，按主题的目标读者来定义。保持基本可读性，不依赖平台特定渲染能力。
